### dende-eventos-madri 🌴📱
#### Este projeto refere-se ao Primeiro Desafio (OAT1) do programa de trainee Dendezeiros, da Dendê SoftHouse. O objetivo é desenvolver a versão inicial do sistema de gerenciamento do aplicativo Dendê Eventos utilizando a linguagem Kotlin.

### 🎯 O Desafio: Gerenciador via Console
#### Nesta etapa, o foco é a lógica de programação e estruturação de dados em Kotlin, seguindo restrições técnicas específicas:
- ##### Linguagem: Kotlin.
- ##### Interface: Operação exclusiva via console (terminal).
- ##### Paradigma: Lógica linear (sem funções ou métodos externos ao fluxo principal).
- ##### Persistência: Dados em memória (Listas/Coleções), sem banco de dados.
- ##### Estruturas: Uso de data class e enum para organização, mas sem lógica interna nelas.

### 🚀 Funcionalidades Implementadas (User Stories)
#### O sistema abrange o ciclo de vida de usuários e eventos, incluindo:
#### Gestão de Usuários: Cadastro de usuários comuns e organizadores (PF/PJ), alteração, visualização e (in)ativação de perfis.
#### Gestão de Eventos: Criação de eventos (Sociais, Acadêmicos, etc.), definição de modalidades (Híbrido, Remoto, Presencial) e regras de estorno.
#### Fluxo de Ingressos: Compra de ingressos (com lógica de eventos vinculados), cancelamento com reembolso e feed de eventos ativos.

### 🛠️ Tecnologias e Restrições
- #### Kotlin
- #### Proibição: Não utilizar funções (fun) além da main, para exercitar a lógica de controle de fluxo e manipulação de coleções de forma bruta.
- #### Versionamento: Git (Branch main).
